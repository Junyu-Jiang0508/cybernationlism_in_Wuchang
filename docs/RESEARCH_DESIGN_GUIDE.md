## 〇、一个先行判断：本研究最深的问题不在技术，在测量

三位评审的意见（层级分类、校准、多层模型、长文本）表面上是技术问题，但它们共同指向
一个更深的设计问题：**当前的测量模型没有完全对齐理论模型。**

- 理论说的是：民族主义话语是**模块化剧目**（modular repertoire）——可拆卸、可重组、
  可跨语境搬运的话语部件（Swidler, 1986 的"工具箱"）。
- 但测量做的是：把每条评论**整体**归入五个互斥类型，然后用类内余弦相似度证明"同质"。

这里有一个概念滑移：**模块化 ≠ 同质化**。同质化说的是"大家说得都一样"；
模块化说的是"大家用同一套零件、以不同组合方式说话"。工具箱理论的核心恰恰是
**可重组性（recombinability）**——如果只测同质性，那么话题狭窄、平台文体惯例、
审查幸存偏差都能产生同样的信号，理论检验就不闭环。

后文所有五个问题的回答，都围绕这个判断展开：**让测量单位从"评论的类型"
下沉到"评论中的话语模块（rhetorical move）"**。这一步同时解决了 Type 1/2 边界
问题的构念根源（见 §三），并把 RQ 从"是否同质"升级为"是否模块化"（见 §五）。

## 一、样本与平台：能否扩大？——能，但按"理论格子"扩，不按数量扩

### 1.1 设计原则

62 万（B 站）+ 4.1 万（知乎）+ 3.3 万（Steam）的语料量已远超统计功效需求。
**约束不是 N，是设计格（design cells）**：每增加一个平台/样本，必须填上一个
理论对比维度的空格，否则只是增加工作量。本研究的理论主张（模块可迁移）
需要的变异维度是：

| 维度 | 现有覆盖 | 空缺 |
|---|---|---|
| 审查制度（强 ↔ 弱） | B 站/知乎（强）vs Steam（弱） | ✅ 已覆盖 |
| 文体/长度（短评 ↔ 长文） | B 站短评 vs 知乎长文 vs Steam 中评 | ✅ 已覆盖 |
| **体裁受控的审查对比** | Steam（游戏评测、无审查）没有国内对照 | ❌ **最值得补** |
| **与既有文献的可比性** | 主流中国民族主义研究几乎全在微博 | ❌ 值得补 |
| **事件泛化（单案例问题）** | 只有《渊虚之羽》一个争议事件 | ❌ **最大外部效度威胁** |
| 平台内部媒介形态 | 只用了评论区，弃用了弹幕 | ⭕ 低成本可补 |

### 1.2 扩展优先级（按理论收益/成本比排序）

**优先级 1：国内游戏平台评测（TapTap / 小黑盒 / WeGame）——体裁受控的审查对比**
Steam 与 B 站的对比混淆了两个变量：审查制度不同 *且* 体裁不同（评测 vs 视频评论）。
补一个"同为游戏评测、但处于国内审查之下"的语料，就能把体裁固定住、
单独识别审查的作用——这是目前设计里唯一能做到"控制变量式"平台对比的机会，
方法论上非常优雅，评审一眼能看懂其价值。

**优先级 2：第二个争议案例——解决单案例设计的外部效度**
目前所有结论都以"渊虚之羽争议"为条件。若模块化是剧目属性而非事件属性，
同一套模块应当在**不同触发事件**中复现。候选：同为游戏领域的《黑神话：悟空》
相关争议（体裁近、便于对照），或跨出游戏域的品牌抵制/影视争议事件
（检验剧目的事件间可迁移性更强，但采集成本高）。设计上只需中等规模
（每案例 5–10 万评论即可），因为目的是"模板复现检验"而非重建全套管线。
**这一条比再加平台更重要**：评审对"单一事件能否代表中国数字民族主义"
的质疑几乎必然出现在全文版评审中。

**优先级 3：微博——与既有文献的锚点**
Schneider (2018)、Repnikova & Fang (2018)、刘海龙（2017）"粉丝民族主义"等
核心文献的经验基础几乎都是微博。补一个微博子样本（不必大）能让你的发现
与整个文献对话："在微博上被反复描述的现象，其结构签名与我们在 B 站发现的一致/不一致"。
没有微博，跨平台声明在文献意义上总缺一角。

**优先级 4（低成本、高辨识度）：B 站弹幕**
弹幕是 B 站的标志性媒介形态：匿名性更强、与视频时间轴同步、篇幅更短。
"同一批视频、同一批用户群体、两种媒介形态"构成天然的平台内对比——
模块在弹幕这种高速、碎片化形态中是否被进一步压缩？这可能是全文版一个
有辨识度的中国平台研究特色章节（西方平台没有等价物）。数据就在已爬的视频列表里。

**不建议扩展的方向（及理由）**
- **抖音**：采集难度与账号风险高；理论上与 B 站同属"强审查+短文本"格子，边际收益低。
- **海外华语圈（YouTube/X/Reddit）**：离散民族主义（diaspora nationalism）是好题目，
  但属于**下一篇论文**的题目（见 §四），塞进本研究会稀释焦点。
- **单纯扩大 B 站样本量**：62 万已够；把预算花在人工标注上（见 §三）。

### 1.3 一个替代性"扩样"思路：时间维度

与其横向加平台，不如纵向拉长：争议**前—中—后**的剧目生命周期
（模板何时出现、何时定型、何时衰减、是否被下一个事件继承）。
现有数据覆盖 2025-07 至 2025-12，事件已进入衰减期；只需对同一关键词池
做一次 2026 年的轻量补采，就能观察"剧目的休眠与复活"——这直接服务于
§五的 RQ（模块的可持续性），而且是纯粹的时间扩展，无新平台成本。

## 二、模型选择：把模型当测量仪器，不当竞赛选手

### 2.1 认识论定位

在 CTAM 的评审语境里，模型是**测量仪器（measurement instrument）**：
它的合法性来自效度链（构念 → 编码本 → 标注 → 模型 → 推断），
不来自 F1 排行榜。Grimmer & Stewart (2013) 的 "validate, validate, validate"
仍是这个领域的第一原则。所有模型决策都应回答一个问题：
**这个选择让哪个推断更可信？**

### 2.2 三层模型架构的角色分工（维持现有两层，增补一层）

| 层 | 角色 | 现状 | 设计建议 |
|---|---|---|---|
| 标注层（LLM） | 生成训练标签 | GPT-4o-mini 单模型 | → **多模型合议**（见 §三），至少加一个开放权重中文模型（如 Qwen 系）保证可复现性——闭源 API 模型会下线，三年后无人能重复你的标注 |
| 放大层（编码器） | 全语料推断 | RoBERTa-wwm-ext | 维持。短文本上编码器微调仍是成本/可复现性最优；Steam 用 XLM-R（已定）；知乎长文的方案已在评审意见中（滑窗/Longformer）。**不必追新模型**——换更大的骨干不会修复测量问题 |
| **模块抽取层（新增）** | 让"模块"本身可见 | 仅有 n-gram 模板检测 | 这是理论要求的一层：黑箱分类器无法展示"模块"，需要能产出**人类可读的模块清单**的方法（近重复检测、模板挖掘、带例句的聚类）。RQ1 的主证据应从这一层出，分类器退居辅助 |

### 2.3 一个容易被忽视的关键点：嵌入模型比分类器更需要稳健性检验

RQ1 的核心证据（类内余弦相似度、聚类富集）**全部依赖嵌入空间的几何**。
不同嵌入模型会给出不同几何——如果"政治化评论类内相似度最高"这个排序
换一个嵌入模型就变了，结论就是伪影。现有的双模型设计（MiniLM + text2vec）
方向正确，但 MiniLM 对中文偏弱；设计上应升级为：
**三个异质嵌入模型（如补一个强中文嵌入 BGE/GTE 系），报告"排序不变性"**
——所有几何结论必须在三个嵌入空间中方向一致才作数。这比任何单模型的
绝对数值都更有说服力，也一劳永逸地堵住"嵌入伪影"的评审质疑。

### 2.4 模型选择的"不做清单"

- 不做 LLM 端到端全语料分类（60 万条 × 多平台：成本高、不可复现、版本漂移）；
- 不为 SOTA 换骨干（评审关心校准曲线和错误分析，不关心你用了什么最新模型）；
- 不用单一模型的输出做任何总体构成声明（校准 + quantification 是前提，评审 R3 已点名）。

## 三、标注策略：编码本重构是全研究杠杆率最高的一步

### 3.1 现有五分类的构念缺陷（这是 Type 1/2 混淆的根源）

现行分类把两个正交维度压进了一个互斥类型系统：

| | 非政治 | 政治化 |
|---|---|---|
| **情感语域** | Type 1（情绪化非政治） | Type 3 的一部分（激愤民族主义） |
| **分析语域** | Type 0/4（评测/中立分析） | Type 2 / Type 4 的边界地带 |

Type 1 vs Type 2 的边界之所以让人类（κ=0.61/0.63）和模型（五分类 F1=0.62）
都难住，是因为**它不是一条边界，而是两个维度的投影**。R3 建议的层级分类
在技术上正确，但更根本的修复在编码本层面：

**方案 A（保守，与既有标注兼容）**：把编码本显式重构为二维——
`politicization`（0–2 有序）× `register`（affective / analytic / mixed）。
现有五类可以映射回这个二维空间，历史标注不作废；层级分类器恰好对应
先判维度一、再判维度二，技术方案与构念结构对齐。

**方案 B（进取，直接测理论构念，推荐作为全文版的核心升级）**：
增加**模块层多标签标注（move-level multi-label annotation）**。
从提交稿的理论段落里，模块清单其实已经写出来了：

- 吹哨/揭发（"发现"辱华实锤）
- 道德仲裁（宣判、定性）
- 国防动员（号召抵制/出征）
- 历史创伤援引（屠杀、国耻叙事）
- 比惨/双标指控（whataboutism）
- 身份忠诚测试（"你是不是中国人"）
- 分析性包装（用理性/考据语气承载上述任一模块——即"爱国迷彩"）

每条评论标注"包含哪些模块"（可多选），而非"属于哪一类"。
这样：(1) 测量单位=理论单位，Swidler 的工具箱第一次被直接操作化；
(2) **可重组性变得可测**——模块共现矩阵、组合多样性、模块间的"语法"
（见 §五 RQ1b）；(3) Type 1/2 边界问题自然消解（一条评论可以既情绪化
又含吹哨模块，不必强行二选一）。
成本控制：模块标注只需在中等规模子样本上做（每平台 1,000–2,000 条），
LLM 初标 + 人工校验，不需要全语料。

### 3.2 LLM–人工分工的设计规范

维持"LLM 初标 → 人工验证"架构，但按 2025–26 年方法文献的新标准升级：

1. **多 LLM 合议替代单模型标注**：2–3 个异质模型（其中至少一个开放权重）
   独立标注，分歧样本进入人工仲裁队列。这同时给出"标注器间信度"——
   LLM 版的 inter-annotator agreement，是 R1 所说 "LLM annotation standards"
   讨论的直接贡献材料。
2. **人工预算花在刀刃上**：不均匀撒人力，按三个分层聚焦——
   (a) 每平台一轮编码本验证（200–300 条双人独立编码，已承诺）；
   (b) **边界定向**：模型置信度最低/多 LLM 分歧最大的样本优先人工；
   (c) 模块层标注的金标准子集（每平台 300–500 条）。
3. **下游推断的偏差修正（最重要的方法升级）**：所有用预测标签做的
   回归/构成估计，改用 **DSL（design-based supervised learning，
   Egami 等）**类框架：以人工金标准子集为锚，对代理标签（surrogate labels）
   的测量误差做设计式修正，得到有效置信区间。这一步直接回应
   "用有噪声的预测变量做推断"这一计算传播学当前最热的方法批评，
   而且让 R3 要求的校准工作从"补救"升格为"完整的测量误差处理框架"。
   总体构成的估计则用 quantification 路线（Hopkins & King, 2010 传播学
   评审最熟悉）。
4. **标注者立场文档化**：中国政治话语的标注高度依赖解释性判断。
   记录编码者背景、分歧仲裁规则；在编码本中保留"构成性模糊"
   （constitutively ambiguous）类别而非强制归类——把提交稿里
   "epistemological ambiguity"的洞见制度化，评审已经表扬过这个姿态。

## 四、后续研究方向：从"一个案例研究"到"一个研究纲领"

以本研究为原点，可延展的方向按"与当前数据的距离"排序：

**近程（现有数据即可支撑，可进全文版或紧接的第二篇）**
1. **剧目生命周期**：模板句的出现—扩散—定型—衰减曲线；跨账号复现的
   时间结构（谁先用、谁跟进）；§一的时间扩采可支撑。
2. **生产者经济学（流量民族主义）**：创作者层数据已有——民族主义内容
   是否是一种**内容策略**？高产账号的模块复用率、词汇多样性、
   发布节奏是否呈现"工业化生产"特征？与 creator economy /
   注意力经济文献对接，"Scripted Rage as *Commodity*" 的标题承诺
   其实要在这一步才真正兑现。
3. **弹幕 vs 评论的媒介形态比较**（§一优先级 4）。

**中程（需新数据或新方法，独立成文）**
4. **审查作为选择机制**：不再把审查当噪声控制，而是当研究对象——
   同一模块在强/弱审查平台上的形态变异（谐音、缩写、图像化）；
   可与 King et al. 的审查理论正面对话。补充采集：同视频评论区的
   时间点快照差分（观测删帖）。
5. **接收端研究**：参与度 ≠ 认同。用调查实验检验"分析性包装的
   民族主义内容是否更具说服力/可信度"——把计算发现（RQ2 的
   参与度溢价）转化为因果机制检验。这是从 CTAM 走向
   political communication 主流的桥。
6. **机器生成的民族主义内容**：2026 年的现实问题——模块化恰恰是
   LLM 批量生成最容易复制的形态。"人写的模板 vs 机器写的模板"
   的可区分性研究，时效性极强。

**远程（研究纲领级）**
7. **模块化民族主义的一般理论**：模块化是中国特色还是数字民族主义的
   一般属性？印度教民族主义 Twitter、俄语爱国 VK、美国文化战争话语的
   跨国比较——把 "modular discourse" 从案例概念推向中层理论
   （Tilly 的抗争剧目 + Swidler 工具箱 + 计算宣传的模板化命题的综合）。
8. **平台治理的规范研究**："爱国迷彩"（analytic 包装的动员内容）
   恰好落在现行内容审核分类的盲区——对平台治理与政策研究的输出。

**两篇论文策略（建议尽早决定）**
评审意见本身已经暗示了分叉：R2 称之为 "a strong methodological piece"，
R1 期待它推动 "LLM annotation standards" 的讨论。建议正式拆分：
- **Paper A（理论+实证）**：模块化剧目的识别、重组与跨平台迁移
  → ICA 2027 / *New Media & Society* / *Chinese Journal of Communication*；
- **Paper B（方法）**：非英语、强审查环境下 LLM 辅助话语测量的效度框架
  （多 LLM 合议 + 校准 + quantification + DSL 全链条）
  → *Computational Communication Research*（该刊正是为这类论文设立的）。
两篇共享数据与管线，但论证独立，互不稀释。RIP 的 35.67 分说明
两个方向都被认可——挤在一篇里反而会复现"评分项 7"的表达问题。

## 五、RQ 制定与理论贴合：从"检验同质性"到"检验模块性"

### 5.1 推导链规范

CTAM 评审读 RQ 的方式是沿着这条链检查断点：

> 理论（Swidler 工具箱）→ 构念（模块化剧目）→ 可观测签名（什么数据模式
> 能且仅能由模块化产生？）→ RQ → 分析设计 → 回到理论的推论

当前 RQ1 的断点在第三环：**语义同质性不是模块化的独有签名**
（话题窄化、文体惯例、审查幸存都能产生它）。修复方式是把签名
细化为模块化的三个不可替代特征：**可识别的部件、结构化的重组、
跨语境的搬运**。

### 5.2 修订后的 RQ 族（含 FINER 检核）

**RQ1a（部件识别，描述性）**
在《渊虚之羽》争议中，民族主义话语由哪些反复出现的话语模块
（rhetorical moves）构成？
*操作化：模块层标注（§三方案 B）+ 模板挖掘；产出带例句的模块类型学。*

**RQ1b（重组结构——模块化的判决性检验，新增，最重要）**
这些模块是以固定脚本整体复现，还是以结构化的组合方式灵活重组？
*操作化：模块共现网络、组合熵/多样性、模块序列模式。判别逻辑：
若只有少数固定组合 → "复制粘贴脚本"（更接近协同行为/水军假说）；
若模块自由重组且组合随语境系统变化 → 真正的"工具箱"使用
（Swidler 的原意）。**这条 RQ 让两个竞争性解释第一次可以被数据区分。***

**RQ1c（跨语境搬运，原 RQ1 的跨平台部分升级）**
同一模块集是否跨平台（审查制度、文体、受众各异）复现，
且形式适应语境而功能保持稳定？
*操作化：模块清单的跨平台匹配（模糊匹配/嵌入近邻）+ 形态变异记录
（如审查规避的谐音变体）；体裁受控对比（Steam vs TapTap，§一）。*

**RQ2a（选择机制，原 RQ2 的理论化升级）**
平台参与度（点赞/回复/转发各自作为不同参与形态）如何随模块类型
与语域包装变化？特别地：分析性语域是否放大民族主义模块的参与度
（"爱国迷彩"机制）？
*操作化：视频层随机截距负二项模型（R3 路线）+ DSL 测量误差修正；
参与度分解（点赞=低成本背书，回复=卷入，转发=扩散）。*

**H2b（方向性假设，多层模型立住后才可提出）**
控制视频层曝露后，分析性包装的民族主义内容比情感外露的
民族主义内容获得更高参与度。
*理论机制：分析语域作为可信度包装（credibility wrapper），
借用理性话语的合法性外衣降低受众的说服抵抗——连接
Papacharissi 的 affective publics 与信源可信度文献。*

**RQ3（生产端，创作者层，可留第二篇）**

参与度回报？（"民族主义作为内容策略"的检验）

### 5.4 Devil's Advocate 检核（三条最强反论及应对）

**反论 1：「你测到的可能是协同行为（水军/国家队），不是文化工具箱。」**
最危险的竞争解释——King et al. (2017) 正好提供了它的理论弹药。
应对：(a) RQ1b 的重组结构分析本身就是判别器（真实用户的工具箱使用
应呈现高组合多样性；协同投放呈现低多样性+时间聚集）；
(b) 账号层辅助信号（账号年龄、发帖间隔、活跃话题广度）做敏感性分析；
(c) 无法完全排除时，诚实地作为 scope condition 写明：
"我们刻画的是话语的生产逻辑，无论生产者是自发用户还是协同行动者——
模块化恰恰使两者难以区分，这本身是一个发现。"

**反论 2：「同质性/模块性可能是审查的幸存者效应。」**
应对：体裁受控对比（Steam vs 国内游戏平台）是核心武器——若弱审查的
Steam 上模块照样复现，幸存偏差解释被削弱；所有跨平台声明保持
"conditional on surviving text"的措辞纪律（提交稿已有，保持）。

**反论 3：「模块清单是研究者建构的，存在确认偏误。」**
应对：模块类型学的产生程序要可审计——自下而上（模板挖掘、聚类例句）
与自上而下（理论推导的候选模块）双向生成，人工双编码检验模块层
的 κ；预留"清单外话语"的开放类别并报告其占比（若很大，理论要修）。

### 5.5 理论定位的最终建议

论文的理论野心分三档，建议瞄准第二档：

1. ~~案例贡献~~（"B 站上的民族主义长这样"）——低于本研究的实际水平；
2. **中层理论贡献（推荐）**："模块化话语生产"作为解释数字民族主义
   规模化与持久性的机制——Swidler 工具箱 + Tilly 剧目 + 计算宣传
   模板化命题的综合，民族主义是案例，机制可移植到其他动员话语；
3. 一般理论（"平台环境系统性地选择模块化政治言说"）——留给 §四的
   远程纲领，本文点到即止（讨论节一段），不要在实证只有一个国家
   一个事件时全面主张。

写作上的对应纪律（针对评分项 7 的 3.67）：每个 RQ 在引言中出现时，
必须能回指到一个具名的理论构念；每个结果小节的首句是一句
plain-language 结论（R1 建议）；"modularity" 在理论节需要一个
操作性定义框（definition box）——R2 明确说了这个概念解释不足。
