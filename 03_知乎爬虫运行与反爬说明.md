## 二、遇到 403 / 验证码 / 安全验证时的应对策略

基于当前项目架构，**只改 Final_Project 下的配置即可**，无需改 MediaCrawler 源码。

### 1. 优先：加大请求间隔（最有效且无成本）

| 现象 | 建议 |
|------|------|
| 403 Forbidden、频繁验证码、要求安全验证 | 拉长每次请求之间的等待时间，降低同一 IP 的请求频率 |

**修改文件：** `Final_Project\01_zhihu_crawler_config.py`

```python
# 默认 2 秒偏激进，知乎反爬严，建议先改为 5~10
CRAWLER_MAX_SLEEP_SEC = 5   # 仍触发可改为 10 或 15，严重时 20~30
```

项目里所有「翻页、拉详情、拉评论」后的等待都会用这个值，调大后整体会变慢，但能明显减少 403 和验证码。

---

### 2. 使用 CDP 模式 + 有窗口（减少被识别为脚本）

| 配置项 | 建议 | 说明 |
|--------|------|------|
| `ENABLE_CDP_MODE` | `True` | 用本机已安装的 Chrome/Edge，环境更接近真人 |
| `CDP_HEADLESS` | `False` | 有窗口、不无头，更不易触发验证 |

在 `01_zhihu_crawler_config.py` 中已默认：

```python
ENABLE_CDP_MODE = True
CDP_HEADLESS = False
```

若你改过源码里的 `base_config.py`，请以本文件为准（本脚本会覆盖为上面这两个值）。

---

### 3. 配置代理 IP 池（同一 IP 被限流时）

当**同一 IP 请求过多**被知乎限流或 403 时，可启用代理，让请求从多个 IP 发出。

**修改文件：** `01_zhihu_crawler_config.py`

```python
ENABLE_IP_PROXY = True
IP_PROXY_POOL_COUNT = 2   # 代理池数量，可按需增加
```

**项目要求：** 需在 MediaCrawler 的 **环境变量或 .env** 里配置代理服务商（如快代理、豌豆 HTTP）。  
- 配置位置：`MediaCrawler-main\MediaCrawler-main\.env`（可复制 `.env.example` 再改）。  
- 具体变量名与格式见：`MediaCrawler-main\MediaCrawler-main\docs\代理使用.md` 以及其内链接的「快代理」「豌豆HTTP」文档。

代理仅负责换 IP；**请求间隔仍然建议保持 5 秒以上**，否则多 IP 也会被单 IP 频率限制拖累。

---

### 4. 降低单次任务量（临时缓解）

若当前一次跑太多（例如 2000 条），可先把上限调小，观察是否还 403/验证码：

在 `01_zhihu_crawler_config.py` 中：

```python
CRAWLER_MAX_NOTES_COUNT = 500   # 先跑 500 条，稳定后再逐步提高
```

---

### 5. 账号与登录态

- **安全验证**：多为账号在知乎侧被风控，需在浏览器/App 里按知乎提示完成验证或换号。  
- **登录态**：项目默认 `SAVE_LOGIN_STATE = True`，会保存登录态，下次运行不必每次扫码；若账号异常，可删除 MediaCrawler 目录下的 `zhihu_user_data_dir` 再重新扫码登录。

---

## 三、推荐调整顺序（按优先级）

1. **先** 把 `CRAWLER_MAX_SLEEP_SEC` 调到 **5～10**，必要时 15～30。  
2. **确认** `ENABLE_CDP_MODE = True`、`CDP_HEADLESS = False`（脚本已默认）。  
3. 仍 403/验证码再考虑：启用代理（配好 .env）+ 适当提高 `CRAWLER_MAX_SLEEP_SEC`。  
4. 仍不行则临时减小 `CRAWLER_MAX_NOTES_COUNT`，或换账号/等一段时间再跑。

所有上述项均在 **Final_Project\01_zhihu_crawler_config.py** 中修改即可，无需改 MediaCrawler 源文件。
